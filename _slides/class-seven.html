APIs

Application Programming Interfaces

It's a way for two applications to talk to each other,
and defines the things each application can request.

You've already been working with one: jQuery!

There's a lot of different kinds of APIs

We care about 2 kinds: library APIs, and web service APIs.

Library APIs

A library is a bunch of code someone wrote and packaged up, that you can use.

You include this library in your project and get to use its functions and methods.

Anytime you use something made available through the library,
you are querying the library API.

Web service APIs

Somewhere out there on a lonely server in the depths of the Internet,
there's a place where some website lives.

It doesn't return human-readable pages! Instead, it returns computer-readable information in response to a request for that information.

If you want that information, you need to give your application
the address to that server, which is often a URL.

Then you write your application to make your request for that information,
and wait for the response from the API.

When people say 'APIs', this is the kind they mean.

AJAX API requests

Have you ever seen a page do something without you refreshing it?
(Like commenting on content or clicking 'like'?)
And if you refresh the page, the changes you made are still there?

Your browser can't remember those kinds of things,
it's the website database remembering them.

Normally, when you want the website database to remember something,
you have to make the change and watch the page refresh.

AJAX changed this completely.

AJAX API requests

When you make an AJAX request,
JavaScript makes a request to the server to update something for you,
without refreshing the page.

When the response comes back, JavaScript can do something with it,
like show you the respnse.
